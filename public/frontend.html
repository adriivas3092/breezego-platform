4. Responsividad & Nav Transformations

Para brindar una navegación nativa en móviles, implementamos transformaciones de interfaces en viewports menores a 900px:

Computadora (Viewport > 900px)

El menú de control lateral se monta fijo a la izquierda (280px), optimizando espacio de lectura horizontal para tablas de fletes.

➔
Celular (Viewport ≤ 900px)

El menú lateral se oculta por CSS y monta un panel circular flotante estilo cápsula en la parte inferior de la pantalla (Uber Eats style) para fácil acceso táctil.

6. Desarrollo Blueprint

La infraestructura cloud de BreezeGo aprovecha microservicios edge distribuidos para escalabilidad automática y latencia menor a 100ms:

🎨 Frontend Engine

Next.js + Tailwind
• shadcn/ui React patterns • Static HTML5 optimized CDN • HSL tailwind colors map

🌐 Edge Routing

Vercel + Cloudflare
• Global static Edge routing • DNS & Cloudflare WAF protected • Sub-100ms load CDN nodes

⚡ Serverless Backend

Supabase Services
• GoTrue user Auth JWT • Websocket Realtime streams • S3 PDF Invoice Buckets

💾 Transaccional DB

Postgres + Upstash
• Postgres RLS row locks • Upstash Redis Rate Limiter • Relational schemas sync

7. Filtros de Seguridad

Filtros de control ejecutados al Edge del servidor para mitigar accesos indebidos y abusos:

🔒 Route Protection Middleware

Valida la cookie del JWT de sesión antes de resolver las rutas protegidas. Si es inválido, redirige automáticamente a /login.

🛡️ Row Level Security (RLS)

Valida que el UUID del usuario autenticado coincida con el user_id del registro. Rechaza lecturas y mutaciones ajenas.

⚡ Upstash Edge Rate Limiting

Rastrea peticiones por IP en Redis. Si un cliente excede 60 llamadas por minuto en OTPSMS o cotizador, bloquea temporalmente con un error 429 Too Many Requests.
